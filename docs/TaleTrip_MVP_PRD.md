# TaleTrip — 黑客松 MVP PRD

**赛道:** QVAC "Unleash Edge AI" · **Mobile**
**日期:** 2026-06-05 · 配套技术方案见《TaleTrip 技术方案 v2》

---

## 1. 一句话
家长在电脑上为孩子生成**专属双语绘本**,孩子在 iPad 上**完全离线、私密**地阅读——故事按这个孩子实时朗读、点词即时翻译发音,还能用相机玩寻物。

## 2. 问题 & 洞察
学龄前(还不识字)孩子的内容,几乎都依赖云、默认会读字、且把孩子数据传到服务器。旅途/睡前/出国这些场景恰恰是"没网 + 不识字 + 家长在意隐私"三重叠加。**把"本地优先"从隐私加分变成刚需**:零网络照常用、孩子数据一字节不出端、内容由家长亲自把关。

## 3. 目标用户 & 场景
- **家长**(创作者):在 MacBook 上几分钟生成一本定制书,亲自预览批准。
- **孩子 4–7 岁**(消费者,不识字):在 iPad 上看图、听讲、点词、玩活动;视觉为主、听觉为辅。
- **场景**:出行(书提前同步好,机上全程离线)、睡前、语言启蒙。

## 4. 赛道契合(Mobile)
fully on-device · private · **travel assistant / language translator / personal tutor** · multimodal(photo→analysis)· creative use of camera · bonus 的 beautiful UX + 真实日常实用性——逐条命中。

---

## 5. MVP 范围(MoSCoW)— 本文件的核心

### ✅ Must(demo 脊柱,必须可用)
- **M1 家长后台(MacBook)**:打字输入主题 → 生成故事文本(LLM)+ 插图(SD)→ **家长预览/编辑/批准** → 打包 StoryPack。
- **M2 内容交付**:StoryPack 同步到 iPad(MVP 用 AirDrop/本地文件即可,P2P 为升级)。
- **M3 iPad 离线阅读**:显示打包绘本 + **LLM 个性化叙述**(填入孩子名/目的地)+ **TTS 朗读**,边看边生成(已实测可行)。轻拍翻页。
- **M4 点词发音**:点高亮词 → TTS 念(英文或当地语)。
- **M5 全程离线 + 隐私**:飞行模式端到端跑通;无网络出站。

### ➕ Should(强 flex,稳则纳入)
- **S1 端侧实时翻译(①)** ✅ **真机验证**:点**任意词** → 端侧翻译 + TTS 发音。**实现:复用常驻 1B LLM + few-shot 严格 prompt**(译文正确,~2.3s/词,零额外模型)——专用 Bergamot NMT 在 A12X 原生崩溃,故走 LLM 路。⚠️ 发音受 TTS 语言闸限制 → 甜区 **EN↔ES**;常见词可配 StoryPack 预烘焙词表即时出。
- **S2 相机寻物(VLM)** ✅ **真机验证**:相机拍 → 端侧 VLM 判"找到没"。**实现:SmolVLM2 500M(multimodal)**,加载 2.1s 无 OOM、单图 ~2.7s、**严格 prompt("不确定答 no")后准确**。Qwen3VL-2B 实测 OOM,故用 500M。自然场景准确性建议实拍再抽检。

### ◻ Could(有余力)
- **C1 涂色**:照片→线稿(边缘检测,非 AI)。
- **C2 多本书库**(MVP 1–2 本演示书即可)。
- **C3 家长后台语音输入主题**(打字已够 MVP)。

### ✗ Won't(本期明确不做)
- 端侧出图 / iPad 上跑 diffusion(4GB 实测不可行)。
- 运行期 P2P 实时出图委托(留作 bonus)。
- 麦克风 / 跟读语音练习(已砍麦克风)。
- 点图中任意物体发音(需区域标注,留后)。
- LoRA / 端侧微调。

## 6. 用户流程

**家长(MacBook,一次性创作):**
打开后台 → 输入主题(如 "Mia goes to Lisbon to meet Grandma")→ 生成故事+插图 → **逐页预览,可改文/换图** → 批准 → 一键打包并发送到孩子 iPad。

**孩子(iPad,离线):**
打开 App → 选书 → 看图 + 听朗读(故事里有自己的名字)→ 点词听发音/看翻译 → 进入活动:相机寻物 / 涂色 → 退出经家长门禁。

## 8. Edge-AI / QVAC 使用
| 运行期(iPad,被评设备) | 构建期(MacBook) |
|---|---|
| LLM 叙述个性化 · TTS 朗读/发音 · **实时翻译(1B+few-shot)** · VLM 相机寻物(SmolVLM2 500M) | STT(可选语音主题)· LLM 故事 · SD 插图 |

→ iPad 上 **3–4 种 QVAC 能力、全离线、多模型编排**(LLM+TTS 实时流水线 + VLM 串行换入),是 demo 的工程亮点。**诚实定位:重生成在创作期 MacBook;运行期可见 AI = 个性化叙述 + 朗读 + 实时翻译 + 相机识别。**

## 9. 非功能需求
- **隐私**:运行期零网络/零遥测;**语音纯内存处理(不落盘);相机帧写 app 私有临时文件 → 推理后即删(VLM 图像输入仅接受文件路径,无法纯内存),不上传/不持久化**;孩子数据仅本地,提供"清除"。
- **内容安全(双层)**:创作期**家长批准**(未批准不推);运行期 LLM 输出经校验,不过回退已批准文本。**绝不把未审核 AI 文本念给孩子。**
- **家长门禁**:PIN 加盐哈希存 Keychain,失败退避。
- **性能(A12X 实测基线)**:LLM+TTS 同驻 4GB 无 OOM;每页生成 ~15.5s < 播放 ~28s;TTS RTF 0.30;VLM(SmolVLM2 500M)加载 2.1s、单图 ~2.7s;点词翻译(1B+few-shot)~2.3s/词。详见《QVAC 端侧基准测试报告》。
- **内存红线**:一次只驻一个大模型;任何屏幕不得后台常驻模型(实测:Home 后台常驻 SD 1.5 曾导致 TTS 假性"OOM 崩溃",且把 LLM 拖慢 ~15×)。单模型实用上限 ~1.5–2GB(SD/Qwen3VL-2B 越线即 jetsam)。

## 10. Demo 定义 & 成功标准
**3 分钟 demo:** 当面切飞行模式 → 家长后台现生成一本"给 Mia 的里斯本之旅" → 发到 iPad → 孩子端离线打开、听到含 Mia 名字的朗读 → 点词实时翻译 → 相机寻物找到"一棵树" → 收尾:"全程零网络、孩子数据一字节没出端,这一切都在一台 2018、4GB 的 iPad 上跑。"

**成功 = ** 离线端到端跑通、无闪崩、个性化可见(换名字叙述就不同)、至少 3 种端侧 AI 现场可见、隐私可自证。
