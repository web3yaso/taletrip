# TaleTrip · 自适应睡眠教练（Sleep Coach）设计

> 2026-06-12 定稿，**未开工**。背景：Bedtime 模式初版被评"设计拧巴、没什么帮助"，
> MedPsy 的真实价值（时差适应）被埋没。本设计将其重构为可执行、可追踪、可自适应的
> 睡眠教练，并给出分阶段落地方案。

---

## 1. 问题诊断（为什么初版不行）

| 问题 | 表现 |
|---|---|
| 服务对象混乱 | 给孩子的哄睡幻灯片上叠着给家长的医学小字，两边都没伺候好 |
| MedPsy 价值被埋 | 3 条建议只露 1 条、小字、无上下文（时差几小时？为何这条？） |
| 输出不可执行 | "保持规律作息"式鸡汤；家长要的是**今晚几点哄睡** |
| 与产品脱节 | 绘本 app 的睡前模式与"今晚的故事"零关联 |
| 一次性 | 计划生成后不随实际执行情况调整 |

## 2. 设计目标

1. **可执行**：输出带时间的逐日行程，每行是"动作 + 时刻"
2. **可追踪**：家长每天 10 秒打卡昨晚实际入睡情况
3. **自适应**：按打卡结果修正当晚目标时间与建议
4. **自动化**：到点 app 主动进入睡眠模式；**可手动退出**（按住 2.5s，已实现）
5. **诚实架构**：小模型不做算术——时间计算确定性，医学文案出自 MedPsy

## 3. 核心形态：逐日时刻表（替代鸡汤句）

```
🏥 Sofia（5 岁）的巴塞罗那时差计划 · 向西 6 小时
─────────────────────────────────────
出发前 2 晚   就寝 21:00（比平时晚 30 分钟）
出发前 1 晚   就寝 21:30
✈️ 飞行日     机上按目的地时间小睡 ≤1h，落地后别补觉
到达第 1 晚   当地 19:30 就寝 · 傍晚多户外光照
到达第 2 晚   当地 20:00 · 白天小睡 ≤45 分钟
到达第 3 晚   恢复正常 20:30 ✓
```

### 可靠性拆分（关键）

- **JS 调度器算骨架**：时差方向/小时数、每天挪 30-60 分钟的标准节奏、光照窗口、
  小睡上限——确定性计算，零算错（小模型算术不可信）
- **MedPsy 填血肉**：逐日医学建议 + 年龄段注意事项（安抚物/屏幕截止/夜醒处理），
  `json_schema` 约束输出（验证过的零脱轨套路）
- 架构叙事：*deterministic scheduler + medical LLM advisor*

### 自适应分支预写（Phase 1 的关键技巧）

生成书时 MedPsy 为**每一天预写两版文案**：

- `advice`：昨晚顺利 → 按表执行
- `adviceIfRough`：昨晚难睡 → 今晚推后 30 分钟 + 加长安抚流程

旅途中 iPad 端只做确定性选择（按打卡结果选分支、±15-30min 调时间），
**所有医学文案仍出自 MedPsy** —— 离线自适应且零端上模型风险。

## 4. 数据模型

```ts
// pack.sleepTips (string[]) 升级为：
pack.sleepPlan = {
  shiftHours: number,          // 时差小时数
  direction: "east" | "west",
  childAge: number,
  baseBedtime: string,         // "20:30" 家长在 Studio 填或默认
  days: [{
    label: string,             // "到达第 1 晚"
    bedtime: string,           // "19:30"
    advice: string,            // MedPsy：顺利分支
    adviceIfRough: string,     // MedPsy：困难分支
  }],
}

// iPad 本地 documents/sleeplog.json：
[{ date: "2026-06-15", asleepAt: "20:10", quality: "smooth"|"ok"|"rough" }]
```

## 5. iPad「Sleep」栏目（Bedtime 两段式重构）

```
🌙 入口（Home 顶栏，保留）
├─ 第一段 · 家长卡「今晚的睡眠计划」
│   📋 逐日时刻表（今晚高亮；今晚时间已按打卡自适应修正）
│   💬 今晚建议（MedPsy 对应分支文案）
│   ☀️ 晨间打卡区：昨晚实际入睡时间 + 顺利/一般/困难 三个大按钮
│   ▶ 「开始哄睡」按钮
├─ 第二段 · 哄睡（点开始后进入，锁定）
│   暗屏 + 白噪声 + 相册慢放（现状保留；后续可选改"轻声念今晚的故事"）
│   tab 栏隐藏（原生锁，已实现）
└─ ✋ 退出：按住 2.5s（已实现并修复）

⏰ 自动进入：app 前台时计时，到达今晚目标时间 → 自动切入第二段
   （demo 演示法：把目标时间设为"2 分钟后"现场触发）
```

### 当前天数定位

“今晚是第几天”由首次打卡日期推算（第一次打卡 = 到达第 1 晚的次日早晨）；
未打卡前默认显示"到达第 1 晚"。家长可在计划表上手动点选当前行（兜底）。

## 6. 展示双落点

- **Studio 生成结果页**（家长在 Mac 前）：完整计划卡挨着二维码——"书给孩子，计划给你"
- **iPad Sleep 栏目**：同一份计划 + 打卡 + 自适应 + 自动进入

## 7. 分阶段落地

### Phase 1（核心，~3-4h，零新模型风险）

1. `studio/medpsy.mjs` 重构：JS 调度器（骨架）+ MedPsy 逐日双分支文案（json_schema）
2. `pack.sleepPlan` schema + Studio 结果页计划卡
3. iPad Sleep 栏目：计划表 + 晨间打卡 + 确定性自适应 + 自动进入计时
4. 哄睡段维持现状（暗屏/白噪声/慢放/长按退出）

### Phase 2（可选炸点，Phase 1 落地后看余量）

**MedPsy 1.7B 装进 iPad**：

- 1.2GB Q4 在单模型红线（~1.5-2GB）内
- 交付方式：**用自家 P2P 从 Mac seed 模型文件给 iPad**（"Mac 把医疗模型本身
  P2P 传给孩子的设备" —— demo 炸点）
- 夜间自适应升级为真·端上 MedPsy 推理（替代预写分支）
- 三个未验证风险：下载时长（~2-5min 一次性）/ A12X 加载 / 1.7B 推理速度（估 10-20s）
- 放 flag 后面，不赌 D4 冻结线

### 明确不做（本周期）

- 锁屏推送提醒（需 expo-notifications + 重建 + 权限；Phase 2 一并评估）
- 睡眠数据图表/周报（超出 demo 范围）

## 8. 评分项映射

| 评分项 | 本设计贡献 |
|---|---|
| Psy 模型创意使用 | MedPsy 从一次性建议 → 纵向自适应教练；Phase 2 端上 MedPsy |
| Complexity & UX | 计划→执行→打卡→修正闭环；自动进入+手动退出 |
| Innovation/P2P | Phase 2 的 P2P 模型分发 |
| 诚实工程 | 确定性调度 + LLM 文案的拆分，README 可写成方法论 |

## 9. 风险与边界

| 风险 | 缓解 |
|---|---|
| MedPsy 双分支文案质量参差 | json_schema 约束 + 长度过滤 + 兜底默认文案 |
| 自动进入打扰（白天误触发） | 仅在目标时间 ±90 分钟窗口内触发；进入前 10s 倒计时可取消 |
| 家长忘打卡 | 未打卡则沿用原计划行，不阻塞 |
| 时区边界（跨日计算） | JS 调度器统一用目的地本地时间，出发前段用家时区 |
